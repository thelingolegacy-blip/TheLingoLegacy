# Activation phase 1–8

This checklist turns The Lingo Legacy static site into a production-ready activation hub on Vercel, then extends it into the next operating layer for content, revenue, community, and metrics.

## Phase 1 — Public HQ foundation

- Publish a root `index.html` so the Vercel production URL has a clear homepage.
- State the launch promise in one sentence: projects, music, merch, community, and founder-led growth.
- Link the homepage to the landing page, asset library, deployment notes, and this activation plan.
- Keep the site static so no build step is required.

## Phase 2 — Offer architecture

- Present the core lanes: projects, music division, merch, community, and future product drops.
- Make the asset library discoverable from the homepage.
- Use the landing page as the conversion surface for future waitlists, drops, and announcements.
- Keep placeholders visible only where a final brand asset or product URL is still pending.

## Phase 3 — Audience engine

- Use the public site as the source of truth for launch updates.
- Add each campaign or drop as a documented lane before promoting it publicly.
- Track launch readiness with a short checklist: message, link, asset, deployment, production verification.
- Keep contact details consistent across README, homepage, and landing page.

## Phase 4 — Production readiness

- Verify the preview deployment is `Ready` before promotion.
- Promote the verified deployment to production on Vercel.
- Confirm the root page, landing page, asset library, and docs links resolve after promotion.
- Record the release in `docs/deploy.md` and keep the README deployment flow current.

## Phase 5 — Content runway

- Define a weekly founder update cadence for public momentum.
- Prepare reusable announcement copy for music drops, merch drops, and project milestones.
- Assign each campaign to one clear lane: awareness, conversion, community, or partner outreach.
- Keep content source files and public links documented before promotion.

## Phase 6 — Revenue pathways

- Map every public page to a next action: listen, buy, join, partner, or learn more.
- Prioritize merch, music, and partner inquiry CTAs for the next launch pass.
- Document offer dependencies such as final product links, pricing, artwork, and fulfillment notes.
- Keep revenue-path placeholders obvious until live URLs are ready.

## Phase 7 — Community capture

- Define audience segments for fans, creators, collaborators, and partners.
- Add signup or inquiry placeholders before connecting the final form provider.
- Document the follow-up loop for new contacts: source, tag, reply owner, and next action.
- Keep community capture aligned with the public launch message.

## Phase 8 — Metrics command

- Define launch KPIs: visits, clicks, signups, merch interest, music engagement, and partner inquiries.
- Review production deployment state before each campaign push.
- Run a weekly operating review covering content shipped, audience growth, conversion signals, and blockers.
- Feed the next backlog from measured campaign outcomes.

## Phase 1–8 completion criteria

- Root homepage exists and is production-safe.
- Landing page reflects all eight activation phases.
- Documentation explains the activation phases and deployment flow.
- Content, revenue, community, and metrics lanes are ready for the next implementation pass.
- No package build step is required for this static site.
