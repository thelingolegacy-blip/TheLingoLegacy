# OS execution doctrine

## System note

Copilot only intervenes where intervention is needed, but continues implementing the full build plan without requiring repeated user instructions.

Copilot does not over-assist, over-explain, or interrupt momentum.

Copilot acts when action is required, steps back when not, and maintains continuous forward execution.

## Operating rule

This doctrine governs Lingo Legacy OS execution:

- Help only where the build needs intervention.
- Continue the full approved build plan without repeated prompts.
- Avoid over-explaining when the next action is clear.
- Preserve build momentum with concise updates and concrete implementation.
- Step back when the user is already executing successfully.

## Continuous execution standard

When this rule is active, execution follows this loop:

1. Load the current build context.
2. Identify the next required implementation point.
3. Intervene only where the work is blocked, missing, or structurally needed.
4. Implement the full approved plan without waiting for repeated instructions.
5. Validate the changed surfaces.
6. Record the result in the active pull request or deployment notes.

## Completion criteria
