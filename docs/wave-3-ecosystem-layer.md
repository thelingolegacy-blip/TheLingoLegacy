# Wave 3 — Ecosystem Layer execution plan

Wave 3 defines the map for assets, nodes, and sync orchestration across Lingo Legacy OS.

## Objective

Prepare the ecosystem model before populating assets or wiring automation.

## Ecosystem lanes

### Global Asset Manager

- Track asset title, type, owner, status, usage rights, and public URL.
- Separate draft assets from production-ready assets.
- Keep final brand assets linked from the asset library.

### Node registry

- Map each ecosystem node by name, lane, owner, status, and dependencies.
- Track project, music, merch, community, partner, and platform nodes.
- Keep node status aligned with activation phase docs.

### Sync engine

- Define what syncs, when it syncs, and what validates success.
- Avoid hidden sync behavior without an owner or rollback path.
- Keep sync output visible in docs or command logs.

## Intervention points

Intervene only when:

- Asset schema is missing.
- Node ownership or dependencies are unclear.
- Sync behavior lacks validation.
- Ecosystem docs drift from public site surfaces.

## Completion criteria

- Asset schema is documented.
- Node registry fields are documented.
- Sync engine rules are documented.
- Wave 3 is linked from the doctrine and README.
