# Sprint activation map

This map turns the OS waves into executable sprint lanes.

## Sprint 1–3 — Foundation lock

- Confirm repo structure, static homepage, landing page, assets, and docs.
- Keep Vercel preview-to-production flow documented.
- Validate HTML, markdown links, JSON maps, and activation markers.

## Sprint 4–6 — Core systems lock

- Maintain XP event schema.
- Maintain wallet ledger schema.
- Maintain identity role registry.
- Add runtime only after schemas stay stable.

## Sprint 7–9 — Command systems lock

- Define Admin Command Center commands.
- Define AI Ops automation boundaries.
- Define Event Bus routing contracts.
- Keep command actions auditable and reversible where possible.

## Sprint 10–12 — Ecosystem lock

- Define Global Asset Manager fields.
- Define node registry ownership and dependencies.
- Define sync engine validation and rollback expectations.

## Sprint 13–15 — Experience and deployment lock

- Define UI shell and OS Home navigation.
- Define Lingo.ai frontend entry points.
- Verify Vercel preview deployments.
- Promote only `READY` deployments.

## Sprint completion rule

A sprint is complete when its docs, scaffolds, links, and validation criteria exist and the active pull request records the result.

## Sprint 15 final lock

Sprint 15 completes the experience and deployment lock by confirming activation docs, deployment runbook, and production readiness are aligned.
